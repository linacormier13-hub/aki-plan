Opportunité spécifique Océan Indien : aucune plateforme de réservation beauté n'est positionnée spécifiquement sur Maurice, La Réunion, Madagascar et les Maldives. AKIBook peut s'imposer comme la référence régionale avant que Fresha ou Treatwell n'y arrivent.
02 Positionnement & différenciation
Le problème des concurrents actuels : Fresha prend 20 % de commission sur les nouveaux clients. Treatwell prend 35 %. Planity coûte ~89 €/mois sans personnalisation. Aucun n'est pensé pour les marchés émergents ni pour être revendu sous marque blanche.

Avantage 1
Zéro commission
Abonnement mensuel fixe. Le salon garde 100 % de ses revenus. Pas de surprise en fin de mois. C'est le principal reproche fait à Fresha et Treatwell.
Avantage 2
Marque blanche complète
Chaque salon achète l'app sous son propre nom et logo. Le client réserve sur "AKI Book" ou "Salon Dupont Book" — pas sur une marketplace concurrente.
Avantage 3
Pensé Océan Indien
Interface en français ET en anglais. Paiement en MUR, EUR, AUD. Support adapté au marché mauricien. Aucun concurrent ne cible cette région.
Avantage 4
Multi-salons natif
Un franchisé AKI gère tous ses salons depuis un seul tableau de bord. Contrairement à Planity qui facture par établissement.

06 Stack technique recommandée
Option A — No-code
Bubble.io + Adalo
Bubble pour la web app, Adalo pour le mobile. Budget : 5 000 – 12 000 €. Délai : 3 à 6 mois. Idéal pour valider le concept rapidement avant d'investir dans du code sur-mesure. Limite : scalabilité à surveiller au-delà de 500 salons.
Option B — Low-code
Supabase + React + Expo
Backend Supabase (open source), frontend React web + React Native mobile via Expo. Budget : 15 000 – 30 000 €. Délai : 6 à 9 mois. Meilleur rapport qualité/prix pour un vrai produit évolutif.
Option C — Sur-mesure
Node.js + Next.js + Flutter
Développement full-stack complet. Budget : 40 000 – 80 000 €. Délai : 12 à 18 mois. Pour une version premium scalable à plusieurs milliers de salons. Nécessite une équipe ou agence.
Ma recommandation : commencer avec l'Option B (Supabase + React + Expo). C'est le meilleur compromis entre vitesse, coût et scalabilité. Un bon développeur freelance full-stack peut livrer en 6 mois pour ~20 000 €. La version No-code est parfaite pour une démo investisseurs mais trop limitée pour un vrai produit SaaS revendable.

09 Prochaines étapes immédiates
Cette semaine
Nommer l'application
Choisir le nom définitif : AKIBook, AKIRdv, BookAKI, ou autre. Vérifier la disponibilité du nom de domaine et déposer la marque en même temps qu'AKI.
Mois 1
Maquette Figma
Faire réaliser les maquettes UI de toutes les écrans clés (15–20 écrans) par un designer UX. Budget : 1 500 – 3 000 €. C'est ce qu'on présentera aux investisseurs et aux premiers salons.
Mois 2
Recruter le développeur
Chercher un développeur full-stack React / React Native sur Malt, Upwork ou via la CCI France-Maurice. Prévoir un test technique avant de signer.
Mois 3
Lancer le MVP
Démarrer le développement du MVP. L'utiliser en premier sur le salon AKI. C'est le meilleur cas d'usage réel pour tester et améliorer le produit avant de le vendre.
